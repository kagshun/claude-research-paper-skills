# 📄 Research Paper Skills for Claude

**Claudeと学術論文を「読む」「書く」ための標準プロトコル**

論文PDFを渡して「分析して」→ 7段階で体系的に読解  
「論文を書きたい」→ 契約駆動型の6段階で構築

どの研究分野でも使えます。

## 🔍 分析プロトコル（読む側）

論文を7段階の標準プロトコルで分析・評価します。

```
Stage 1: メタデータ抽出
Stage 2: 研究の位置づけ（Research Gap → Motivation → Contributions）
Stage 3: 手法の解剖（数式には直観的説明付き）
Stage 4: 結果の精査（統計的有意性 vs 実用的意義の区別）
Stage 5: 批判的評価（内的妥当性・外的妥当性・論理的一貫性）
Stage 6: 文脈化と応用可能性
Stage 7: 総合評価サマリー（一言要約・最大の強み・弱み・推奨アクション）
```

**論文タイプ別に重点が自動調整されます：**
- RCT → 結果と批判的評価に重点（CONSORT準拠確認）
- AI/CS系 → アーキテクチャとablation studyに重点
- 質的研究 → 理論的飽和とリフレキシビティに重点
- 系統的レビュー → PRISMA準拠と異質性評価に重点

➡️ [分析プロトコルの全文を見る](skills/analysis-SKILL.md)

## ✍️ 執筆プロトコル（書く側）

「論文契約（Manuscript Contract）」を最初に策定し、セクション間の一貫性を構造的に保証しながら執筆します。

```
Phase 0: 執筆モード判定
Phase 1: 論文契約の策定 ← 最重要
         ├── RQ確定（PICOS形式）
         ├── 貢献リスト（C1, C2, C3...）
         ├── 依存関係マップ（Cn → Mn → Rn → Dn）
         └── 視覚要素の登録
Phase 2: 先行研究の位置づけ（Gap構造化・引用マップ）
Phase 3: セクション別執筆（毎回契約チェック付き）
Phase 4: 全体整合性レビュー（査読者シミュレーション3視点）
Phase 5: 査読対応（Response Letterテンプレート付き）
Phase 6: フォーマット仕上げ
```

**核心は依存関係マップ：**
```
C1（貢献）→ M1（Methods内の裏付け）→ R1（Results内の実証）→ D1（Discussion内の考察）
```
Introductionで述べた貢献が、Methods → Results → Discussion で一貫して追跡されます。

➡️ [執筆プロトコルの全文を見る](skills/writing-SKILL.md)

## 🚀 インストール方法

### 方法1: メモリに登録する（最も簡単・おすすめ）

Claude（claude.ai）との会話で、以下をそのまま送信してください。

**分析プロトコル：**

> 以下をメモリに登録してください：
> 
> 【研究論文分析プロトコル】論文の分析・解説依頼時は以下の7段階標準プロトコルに従う：①メタデータ抽出（著者・掲載誌・日付）②研究の位置づけ（Research Gap・Motivation・Contributions）③手法の解剖（研究デザイン・理論的枠組み・アーキテクチャ、数式には直観的説明を添付）④結果の精査（定量結果・効果量・統計的vs臨床的意義の区別）⑤批判的評価（内的妥当性・外的妥当性・論理的一貫性・著者が認めた限界+未言及の限界）⑥文脈化と応用可能性（ユーザーの研究プロジェクトとの接続）⑦総合評価サマリー（一言要約・最大の強み・最大の弱み・推奨アクション）。論文タイプに応じて重点調整（RCT→Stage4-5、AI系→Stage3-4、質的研究→理論的飽和・リフレキシビティ）。日本語では専門用語に英語併記。

**執筆プロトコル：**

> 以下をメモリに登録してください：
> 
> 【論文執筆プロトコル】Claudeとの論文執筆時は以下の6段階で進行：Phase0)執筆モード判定（フルビルド/セクション執筆/リライト/査読対応/フォーマット調整）→Phase1)論文契約策定（RQ確定[PICOS形式]・貢献リスト[C1-Cn]・セクション依存関係マップ[Cn→Mn→Rn→Dn]・視覚要素登録・投稿先制約）→Phase2)先行研究の位置づけ（Gap構造化・引用マップ構築）→Phase3)セクション別執筆（各セクションで契約充足チェック実施）→Phase4)全体整合性レビュー（契約充足・クロスセクション一貫性・査読者シミュレーション3視点）→Phase5)査読対応→Phase6)フォーマット仕上げ。最重要原則：Phase1の契約策定が全体の土台。科学的事実は捏造せずデータ不足箇所はプレースホルダーで示す。

登録後は、新しいチャットを開いても自動的にプロトコルが適用されます。

## 💡 使い方の例

### 論文を分析する
```
（PDFを添付して）
「この論文を分析して」
```
→ 7段階プロトコルで体系的に分析

### 論文を書く
```
「○○についての論文を書きたい。対象は△△で、□□という手法で調べた」
```
→ Phase 1（論文契約策定）から対話的にスタート

### 分析 → 執筆の連携
```
1. 関連論文を分析プロトコルで読み解く
2. 分析結果を執筆プロトコルの Phase 2（引用マップ）に反映
3. Discussion での先行研究との比較にそのまま活用
```

## ⚙️ カスタマイズ例

メモリ登録版はコンパクトなので、自分の研究分野に合わせて調整できます。

**臨床研究者の場合：**
```
…論文タイプに応じて重点調整（RCT→CONSORT準拠確認とITT分析、
観察研究→STROBE準拠、質的研究→COREQ準拠）…
```

**情報科学研究者の場合：**
```
…論文タイプに応じて重点調整（AI系→アーキテクチャ・ablation・計算コスト・再現性、
システム論文→スケーラビリティ・レイテンシ・ユーザースタディ）…
```
