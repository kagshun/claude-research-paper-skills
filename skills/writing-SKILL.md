# Research Paper Writing Protocol

Claudeと協働して学術論文を段階的に構築するためのスキル。

## 設計思想

論文執筆における最大の失敗モードは**セクション間の不整合**である。Introductionで述べた貢献がMethodsで裏付けられない、Resultsで検証されない、Discussionで回収されない——この「構造的ドリフト」がLLMとの協働では特に起きやすい。

本プロトコルは、執筆の最初に**論文契約（Manuscript Contract）**を策定し、全セクションでこの契約を参照・検証しながら進めることで構造的ドリフトを防ぐ。

---

## Phase 0: 執筆モード判定

ユーザーのリクエストに応じて適切なモードと開始Phaseを選択する。

| モード | トリガー例 | 開始Phase |
|---|---|---|
| フルビルド | 「論文を一から書きたい」 | Phase 1 |
| セクション執筆 | 「Methodsを書いて」 | Phase 3（契約未策定ならPhase 1から） |
| リライト・改善 | 「このドラフトを改善して」 | Phase 4 |
| 査読対応 | 「リバイズして」「査読コメントに対応して」 | Phase 5 |
| フォーマット調整 | 「○○ジャーナルの形式に直して」 | Phase 6 |

---

## Phase 1: 論文契約（Manuscript Contract）の策定

**最も重要なPhase。ここが全体の土台になる。**

ユーザーとの対話を通じて以下を確定し、明文化する。

### 1-1. Research Question（RQ）の確定

PICOS形式で精緻化する：
- **P**（Population）：誰を/何を対象にしているか
- **I**（Intervention / Exposure）：何をするのか、何を調べるのか
- **C**（Comparison）：何と比較するのか
- **O**（Outcome）：何を測定するのか
- **S**（Study Design）：どうやって調べるのか

RQが曖昧な場合、上記を順に質問して明確化する。最終的に1〜3文に凝縮する。

### 1-2. 貢献リスト（Contribution Claims）

この論文が主張する貢献を列挙する（通常2〜4項目）：
```
C1: [第一の貢献]
C2: [第二の貢献]
C3: [第三の貢献]
```

### 1-3. セクション依存関係マップ

各貢献がセクション間でどう展開されるかを追跡する：
```
C1 → M1（Methods内の裏付け） → R1（Results内の実証） → D1（Discussion内の考察）
C2 → M2 → R2 → D2
C3 → M3 → R3 → D3
```

**契約ルール：** 各Cnは、対応するMn・Rn・Dnがすべて存在しなければならない。一つでも欠けていたら不整合として検出する。

### 1-4. 視覚要素の登録（Visual Registry）

必要な図表を事前に登録する：
```
Fig.1: [内容] → 配置: Methods
Fig.2: [内容] → 配置: Results
Table 1: [内容] → 配置: Results
```

全図表は本文中で必ず参照されなければならない。

### 1-5. 投稿先の制約

ターゲットジャーナルが決まっている場合：
- ジャーナル名
- 構成要件（IMRAD、特殊セクション等）
- 語数制限（Abstract、本文）
- 図表数の上限
- 参考文献スタイル
- 必須セクション（Data Availability、Ethics、COI 等）

必要に応じてweb検索でAuthor Guidelinesを確認する。

### 契約の確認

Phase 1完了時に統合した「論文契約書」を提示し、ユーザーの合意を得る。合意後、この契約が以降すべてのPhaseの基準となる。

## Phase 2: 先行研究の位置づけ（Positioning）

### 2-1. Research Gapの構造化

3層で整理する：
- **What is known：** 既存研究が達成していること
- **What is not known：** まだ達成されていないこと
- **Why it matters：** なぜそのギャップが問題なのか

### 2-2. 引用マップの構築

主要文献を論文中での機能別に分類する：
- 背景・文脈の確立に使う文献
- Research Gapの根拠として使う文献
- 方法論的先行事例として使う文献
- 結果の比較対象として使う文献
- 理論的枠組みの出典

各文献は論文中での役割を明確にし、「並べただけ」の引用を避ける。

### 2-3. Related Work の構成設計

テーマ別またはアプローチ別にグルーピングし、各グループが最終的にResearch Gapへ収束する構造にする。時系列の羅列ではなく、「なぜこの研究が必要か」を論証するナラティブとして構成する。

---

## Phase 3: セクション別執筆

各セクションを契約に基づいて執筆する。**セクションを1つ書くたびに、契約との整合をチェックする。**

### 3-A. Abstract

**執筆タイミング：** 全セクション完成後に最後に書く。

Structured Abstractの場合の構成：
- **Background:** Research Gapの1〜2文要約
- **Objective:** RQの凝縮形
- **Methods:** 研究デザイン・対象・主要手法
- **Results:** 主要結果（数値を含む）
- **Conclusions:** 主張の帰結と意義

チェックルール：
- Abstractの内容は必ず本文に存在する（本文にない情報を書かない）
- 数値は本文と完全一致させる

### 3-B. Introduction

読者を「なぜこの研究が必要か」→「この論文は何をしたか」まで導く。

推奨構成（逆三角形）：
1. 広い文脈の提示（1〜2段落）
2. 既存研究の要約とGap（2〜3段落）
3. 本研究のアプローチ（1段落）
4. 貢献の明示（1段落）：C1, C2, C3...

**契約チェック：**
- [ ] C1〜Cnがすべて言及されているか
- [ ] Research Gapが明確に定義されているか
- [ ] 後続セクションへの期待値が適切に設定されているか

### 3-C. Methods

第三者が再現できる詳細さで記述する。

構成要素（研究タイプに応じて取捨選択）：
- Study Design：研究デザインの明示と正当化
- Participants / Data：選定基準、サンプルサイズ、募集方法
- Intervention / System：介入内容またはシステム構成
- Measures / Outcomes：測定指標の定義と方法
- Procedures：データ収集手順
- Analysis：分析手法、統計モデル、有意水準
- Ethical Considerations：倫理審査、同意取得

**契約チェック：**
- [ ] 各Cnに対応するMnが記述されているか
- [ ] Visual Registryの図表が適切に参照されているか
- [ ] 再現に必要な情報は十分か

### 3-D. Results

得られた結果を**解釈なしに**提示する（解釈はDiscussionで行う）。

執筆原則：
- Methodsで記述した分析の順序に沿って提示する
- 各結果と対応するRQ/仮説の関連を明示する
- 図表を先に参照し、本文で要点を言語化する
- 主要結果 → 副次結果 → 探索的結果の順で構成する
- ネガティブな結果も隠さない

**契約チェック：**
- [ ] 各Mnに対応するRnが提示されているか
- [ ] 全図表が参照されているか（参照順序も正しいか）
- [ ] Introductionで設定した問いへの回答が含まれているか

### 3-E. Discussion

結果を解釈し、文脈に位置づけ、限界を認め、意義を主張する。

推奨構成：
1. 主要結果の再述と解釈（1〜2段落）
2. 先行研究との比較（2〜3段落）
3. メカニズムや理論的説明（1〜2段落）
4. 臨床的・実用的含意（1段落）
5. Limitations（1〜2段落）：結果の解釈にどう影響するかを述べる
6. Future Directions（1段落）
7. Conclusion（1段落）

**契約チェック：**
- [ ] 各Rnに対応するDnが議論されているか
- [ ] Introductionで述べたGapが回収されているか
- [ ] Limitationsが誠実に記述されているか
- [ ] 結論がデータに支持される範囲を超えていないか

## Phase 4: 全体整合性レビュー

ドラフト完成後、3つの観点で検証する。

### 4-1. 契約充足チェック

```
C1 → M1 → R1 → D1 : [充足 / 不足箇所]
C2 → M2 → R2 → D2 : [充足 / 不足箇所]
Fig.1 → 本文参照あり : [確認 / 未参照]
Table 1 → 本文参照あり : [確認 / 未参照]
```

### 4-2. クロスセクション一貫性チェック

| チェック項目 | 内容 |
|---|---|
| 用語の統一 | 同じ概念に異なる用語を使っていないか |
| 数値の一致 | Abstract・Results・Discussion間で数値は一致するか |
| 時制の適切さ | Methods=過去形、確立された知見=現在形 等 |
| 主張の整合 | IntroductionとConclusionが対応しているか |
| 図表参照 | 全図表が参照され、順序は正しいか |
| 引用の機能 | 各引用は論証上の役割を果たしているか |

### 4-3. 査読者シミュレーション（3視点）

**Reviewer 1（方法論重視）：**
- サンプルサイズは十分か？パワー分析は？
- バイアスのリスクは管理されているか？
- 統計手法の選択は妥当か？

**Reviewer 2（新規性重視）：**
- 既存研究との差別化は明確か？
- 貢献はincrementalでないか？
- この研究が存在することで何が変わるか？

**Reviewer 3（実用的意義重視）：**
- 統計的有意性と実用的意義は区別されているか？
- 実世界への示唆は具体的か？
- エンドユーザーにとってのベネフィットは明確か？

各Reviewerの想定コメントとその対応案を提示する。

---

## Phase 5: 査読対応（Revision）

### 5-1. コメントの分類
- **Major：** 追加分析、方法論の変更、構造的改訂が必要
- **Minor：** 表現修正、引用追加、図表改善で対応可能
- **Editorial：** 誤字脱字、フォーマットの修正

### 5-2. Response Letterのテンプレート
```
Reviewer #N, Comment #M:
[コメントの要約]

Response:
[対応内容の説明]
[原稿のどこをどう変更したか]

Revised text (p.X, lines Y-Z):
[変更後のテキスト]
```

### 5-3. 改訂カスケードの管理
一箇所の変更が他セクションに波及する場合がある。改訂時は契約を更新し、影響が及ぶ全セクションを再チェックする。

---

## Phase 6: フォーマット・仕上げ

### 6-1. ジャーナル固有の要件適合
投稿先が確定したら：
- Author Guidelinesをweb検索で確認
- 語数制限、構成要件、参考文献スタイルへの適合
- 必須セクション（Data Availability、COI、Funding 等）の確認
- 報告ガイドライン（CONSORT、STROBE、PRISMA 等）への準拠

### 6-2. 英文チェックリスト
英語論文の場合の最終確認：
- 冠詞（a/the）の適切性
- 主語と動詞の一致
- 受動態の過剰使用
- Hedging表現の適切さ（may, suggest vs. demonstrate, confirm）
- 段落間のトランジション

## 実行上の原則

### Claudeの役割と限界
- Claudeは執筆支援ツールであり、共著者ではない。最終的な学術的判断と責任はユーザーにある
- 科学的事実を捏造しない。データがない部分は `[ここにデータを挿入]` 等のプレースホルダーで明示する
- ユーザーが提供した情報の範囲内で執筆し、推測で科学的主張を追加しない

### 進め方
- Phase 1の契約策定を飛ばさない。ここが崩れると全体が崩れる
- 各セクション執筆後に契約チェックを実行する
- 大幅な方針変更がある場合は契約を更新してから進む

### 分析プロトコルとの連携
research-paper-analysisスキルで分析した論文の結果を以下に活用できる：
- Phase 2の引用マップへの反映
- Phase 3-Eの先行研究との比較
- 分析で抽出したResearch Gapを自身のGap定義の参考にする
